# Milestone 3 Persistent World Definition

## Purpose
This document defines what persistent world should mean in Margot after the personal, household, and civic baselines are already proven, so later Milestone 3 planning has one narrow target instead of drifting into broad simulation or premature expansion.

## What Persistent World Means Here
Persistent world in this project means visible consequence that stays true after the moment that created it, can be revisited in place, and still matters to later choices. It does not mean the world feels busy by running many background systems. It means prior care, neglect, repair, or contribution leaves a durable mark that players can see and depend on later.

## What Persistent World Is Not
- not a giant always-running village simulation
- not fake NPC noise meant to imply life without meaningful consequence
- not timer spam, upkeep spam, or routine churn for its own sake
- not vague world "activity" that players cannot point to
- not broad system sprawl added to make the game seem deeper than it is
- not a license to start town-scale AI, general procedural world logic, or milestone-wide implementation

## The 3 Core Dimensions
### Persistent place
One known place should keep a changed condition that players can return to and recognize later.

### Persistent role
One known specialist or place-role should remain meaningfully tied to that place condition, not through broad NPC memory, but through a stable local responsibility or relationship.

### Persistent dependency
Later use, planning, or judgment should depend on whether that prior place condition still holds, so the world carries consequence forward instead of resetting to neutral.

## What Milestone 3 Should Prove
Milestone 3 should prove the smallest new truth not already proven by Milestones 1 and 2: one already-known village place can hold a durable shared consequence that remains visible later, still matters to an already-known role, and creates one ongoing dependency players can understand. Milestone 1 already proved productive ownership and one civic world change. Milestone 2 already proved explicit household meaning. Milestone 3 should prove that the world itself can keep local meaning over time without turning into broad simulation.

## Candidate Milestone 3 Shapes
1. One specialist-linked existing-village place gains a durable ready or restored condition that players can revisit later, and one already-known specialist relationship depends on that condition.
2. The bridge gains one narrow bridge-adjacent persistent consequence inside the already-known village footprint, so the shared civic win has one small local afterlife beyond simple completion.
3. Current household meaning gains one home-facing persistent-place expression, so house-ready consequence is revisitable as place truth instead of reading only as pantry status text.

## Best Milestone 3 Shape
The best Milestone 3 shape is one specialist-linked existing-village place with a durable ready or restored condition. It should start inside the already-known village footprint and reuse an already-known place and specialist relationship if possible, not default to new-area expansion. This is the strongest small proof because it makes persistent world meaning local, visible, and revisitable while staying compatible with current baselines and AI-assisted implementation discipline. It also proves more than a one-time unlock without widening into a new district, broader household framework, or vague world simulation.

## What Must Stay Unchanged
- personal inventory, pies, coins, and `asset/apple_tree` ownership or yield remain personal authorities
- household scope remains explicit and narrow; this milestone must not backdoor shared spending, shared tree ownership, or household `item/pie`
- civic bridge funds, stages, and unlocks remain civic authorities
- personal, household, and civic state must not collapse into one mixed world-consequence path
- stateless Milestone 1 NPC reflection must not quietly turn into broad NPC simulation or memory sprawl

## State / Save Boundary Recommendation
Derived meaning should stay preferred by default. Persistent role should derive from the existing place and specialist relationship already defined in data. Persistent dependency should derive from explicit world truth that players can revisit, not from hidden timers or ambient activity counters. This pass does not authorize a new saved shared place-condition field. Milestone 3 may justify one narrow saved place-condition field later only if derivation alone proves insufficient, and only as an explicit single-purpose condition attached to one known place rather than as a vague authority blob. Do not create a world-activity ledger, timer grid, fake background-sim state, or mixed personal-household-civic persistence layer.

## Smallest Good Milestone 3 Expression
Use one already-known village work place, preferably tied to one already-known specialist, and let that place hold one durable ready or restored condition that remains visible on return and matters to later use there.

## Deferred To Later
- new district expansion as the default proof of persistent world
- broad procedural village simulation
- general NPC routine, memory, or AI systems
- timer-heavy maintenance webs
- extra authority layers or world-state blobs
- household scope expansion beyond the frozen baseline
- additional civic rewrites, institution systems, or town-wide management

## Recommendation
Treat Milestone 3 as the smallest proof that one known village place can keep meaningful consequence over time. Start with one existing place, one existing specialist relationship, and one clear dependency that remains true later. If that works, later expansion can build from a real persistent-world foundation instead of from simulation theater.

## Final Rule
Persistent world means a place still matters tomorrow, not that everything runs all the time.
