# Production Docs

## Purpose
These docs control change during inception and early pre-production. Use them to decide what to build next, how to keep saves stable, and how to avoid drifting away from the canon.

## How To Use Them
- `roadmap.md`: milestone intent, banned scope, and success proofs
- `backlog.md`: actionable work grouped by milestone
- `decision_log.md`: concise accepted decisions with dates
- `id_policy.md`: stable naming rules for long-lived content
- `save_compatibility_policy.md`: rules for save shape changes, migrations, and deprecations
- `content_schema_reference.md`: compact design-facing reference for content tables

## Working Rule
Canon decides meaning. Design chooses the current expression. Production controls change so the repo can evolve without constant rewrites or save pain.
