# Post-Task Documentation Sync

## When To Use
Use this skill after any substantial design, production, architecture, implementation, milestone-planning, or repo-coherence task.

## Required Read Set
- `README.md`
- `AGENTS.md`
- `docs/README.md`
- `docs/production/current_state.md`
- `docs/production/README.md`
- `docs/production/backlog.md`
- `docs/production/roadmap.md`
- `docs/production/risk_register.md`
- `docs/production/development_operating_model.md`
- `docs/production/decision_log.md`
- directly impacted design or production docs from the completed task

## Checklist
1. Check reading order and navigation links.
2. Check current milestone, current priority, and next approved baby steps.
3. Check active risks and reinforcement language.
4. Check backlog, roadmap, and decision log for real state changes.
5. Check AGENTS read paths and output contract.
6. Check for stale paths, stale references, or missing new docs.
7. Update only the files that are genuinely affected.

## Required Output
- concise summary of docs updated
- reviewed-file checklist
- changed versus unchanged per file
- most important repo-state update
- any new durable AGENTS rule added
